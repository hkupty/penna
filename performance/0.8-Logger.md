# 0.8 Logger Performance

This benchmark follows the same structure as the [previous one](performance/0.6-Logger.md), but with a few notable changes:

- The time unit is now `µs`, instead of `ns`;
- Logstash Logback encoder was removed since logback now offers a native json appender;
- Log4j was added as a popular slf4j backend;
- The UUID argument was replaced by a static value not to skew the results.


The test that generated this results [can be found here](https://github.com/hkupty/penna/blob/48d2dd2f001b46339ce6ac436df2e286be2f06f0/penna-perf/src/main/java/penna/core/logger/LoggerPerformanceTest.java).

## Test cases

The test cases are:

- `simple`: A simple log message;
- `modest`: The previous case with an added `Marker`;
- `moderate`: The previous case with an added `MDC` key-value pair and an argument formatted;
- `large`: The previous case with an added exception;

## Results

Below you'll find two charts. The simple result shows log write latency in µs and allocation rate.
The full result will also show perf statistics.

### TL;DR

![image](https://github.com/hkupty/penna/assets/2695819/ac8289a2-5137-474b-817d-c9bd53752073)
![image](https://github.com/hkupty/penna/assets/2695819/b6a80d8b-3b36-40ac-a3d1-990db99c066c)
![image](https://github.com/hkupty/penna/assets/2695819/e96678ac-af2e-4803-b8ea-9734dca1c50f)

- Penna is 3.7x-5.9x faster than Logback and 3.7x-26.4x faster than Log4j;
- GC time is 2x-61x smaller than Logback and 1.3x-42x smaller than Log4j;
- Memory consumption/operation is 27.7x-2558.5x smaller than Logback and 6.2x-1277.6x smaller than Log4j;
